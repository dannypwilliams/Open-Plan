# OPEN PLAN — Decision Log

## Release decisions

- Preserve Unity's FBX axis-conversion transform on a visual child, with gameplay components on an unscaled wrapper. This is the stable boundary between Blender-authored meters and Unity importer scale.
- Use deterministic lightweight steering instead of a NavMesh. Twelve workers and a deliberately open one-floor layout make it sufficient for prototype scope, while stuck recovery prevents permanent stalls.
- Generate TMP font assets dynamically from bundled Inter and ship canonical TMP essentials so UI has no machine font dependency.
- Build simulation and presentation procedurally from two tiny bootstrap scenes. This keeps scene serialization stable while source systems remain modular and testable.
- Record release media through a deterministic real-time tour using the same camera, HUD, workers, and management APIs as ordinary play. No separate marketing scene exists.
- Score readiness at 79/100 rather than treating test success as commercial polish; animation and audio depth remain the clearest limitations.

## D001 — Reproducible authored scene generation

The office scenes, shared materials, prefabs and settings are generated by a checked-in Unity editor pipeline. This makes the prototype rebuildable from source and avoids a fragile sequence of undocumented editor clicks.

## D002 — FBX contract

Blender generates low-poly, beveled foundational meshes in background mode. Unity consumes copied `.fbx` exports only; `.blend` sources are preserved but never imported by Unity.

## D003 — Transform-driven characters

Workers use a modular imported mesh hierarchy and lightweight transform animation. The prototype favors readable bob, typing, drinking, talking, reactions and box-carrying over a costly skeletal facial rig.

## D004 — Deterministic simulation

Worker choices use a single seedable random service and bounded decision intervals. Needs and cooldowns explain state transitions and prevent rapid oscillation.

## D005 — One-floor focus

All effort stays on a 30 m by 22 m office diorama. Meeting, reception, utility, coffee, water, manager and desk zones are visual/behavioral parts of that one floor only.
